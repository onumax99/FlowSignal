# FlowSignal 予測アプローチ設計メモ（lead-lag / 精度向上）

> 銘柄間の関連性（リードラグ）を軸にした予測の狙いと、予測精度を上げるための推奨をまとめた検討メモ。
> 関連: 要件定義 [requirements.md](requirements.md) / ステータス [STATUS.md](STATUS.md)

- 作成: 2026-06-16
- ステータス: **検討メモ（未確定）**。本メモの内容は requirements.md / STATUS.md の現行 M2 設計にはまだ未反映。合意後に反映する。

---

## 0. このメモの結論（要約）

- やりたいこと =「**ソフトバンクが動くと翌日 TOTO が動く**」のような**銘柄間リードラグ**を見つけて予測に使う。狙い自体は妥当。
- ただし**現行の M2 設計では拾えない**（自分のテクニカル＋市場指標だけで、他の個別銘柄の動きが特徴量に入っていない）。
- 日次・大型株のペア間 lead-lag は**弱く不安定**になりがち。単独の主役にせず、**クロスセクション予測＋イベント駆動（LLM）**の枠組みの中の「有力な一特徴」と位置づけるのが現実的。
- 精度を上げる一番効くレバーは、モデルいじりではなく **①棄権（高確信だけ予測）②相対（クロスセクション）予測 ③イベント駆動の新情報（LLM×開示）**。

---

## 1. 現設計とのギャップ

現行 M2（[STATUS.md](STATUS.md) §6）の特徴量は、各銘柄について
- **自分自身のテクニカル**（リターン / 移動平均乖離 / RSI / MACD / ボラ / 出来高）
- **市場全体の指標**（日経・TOPIX・USDJPY・米指数・VIX）

のみ。学習行は `(銘柄, 日付)` 単位で、**他の個別銘柄の動きを表す列が 1 つも無い**。

→ 「A が t に動いた → B が t+1 に動く」を表す材料がモデルの入力に存在しないため、**原理的に lead-lag を学習できない**。全銘柄を pooled で学習しても、行が別々で横断する特徴が無いので関連は結べない。

## 2. lead-lag を実現するための設計要素

| # | 要素 | 内容 |
|---|---|---|
| ① | **他銘柄のラグ付きリターン**（核心） | ターゲット B の t+1 を予測する行に `他銘柄 A の r(t), r(t-1), …` を入れる。15銘柄×数ラグなら次元も現実的 |
| ② | **lead-lag の発見工程** | ラグ相互相関 / グレンジャー因果でペアの先行関係を炙り出す → 根拠提示（FR-6）と特徴選択に使う。学習後は feature importance で「実際に効いた先行銘柄」を確認 |
| ③ | **市場中立化**（重要な落とし穴回避） | 大型株は市場共通要因で同時に動くため、素のリターンで lead-lag を取ると「両方が TOPIX に連動しているだけ」を拾う。各銘柄リターンを TOPIX 等で回帰した**残差**で lead-lag を見ると、銘柄固有の先行関係が出やすい |
| ④ | **モデル構造** | 「何が B を先導するか」発想なら **per-target（銘柄別）モデル**が素直（15個でも軽い）。pooled＋銘柄ID＋横断ラグでも可だが per-target を推奨 |
| ⑤ | **リーク防止（一段厳格に）** | 他銘柄特徴も t 以前のみ（t の終値時点で既知）。どの銘柄も t+1 は絶対に使わない。**日付境界での walk-forward** が一層重要 |

### 2.1 「見かけの精度」を生むリーク/スヌーピング（本メモが推す手法ほど要注意）

本メモの手法は強力だが、その分だけ過学習・データスヌーピングで精度を錯覚しやすい。設計に落とす前に必ず明記する:

1. **市場中立化の β もリーク源（②③）**: TOPIX で回帰した残差を使う場合、**β を全期間 OLS で推定すると未来情報が漏れる**。β は rolling / expanding（過去のみ）で推定する。残差特徴を t 以前に揃えるだけでは β の leak は防げない。
2. **lead-lag 発見はスヌーピング（②）**: 15×15ペア×複数ラグの Granger / 相互相関は多重比較で偽陽性が大量に出る。全データでペアを選んで同じデータで評価すると選択リーク。**発見は train fold 内のみ**で行い（fold をまたいで固定しない）、多重比較補正するか「探索・可視化用」と割り切る。feature importance での事後確認も test を見ない。
3. **較正・棄権閾値は過去/検証データで決める（→ §4①）**: 確率較正は test ではなく過去の較正区間で fit（時系列順守）。小標本では **isotonic は過学習しやすく Platt を既定**に。「確信度上位だけ予測」の閾値も train/validation で決める（test 上で最良閾値を選ぶと coverage×精度が楽観バイアス）。
4. **per-target × クロス特徴 × 薄い標本の過学習（①④）**: 銘柄別モデル（5年で約1200行）に 15銘柄×数ラグを足すと次元過多。ラグ数を絞る／正則化前提で。クロスセクションも銘柄15では断面が薄い（§4⑤「銘柄数を増やす」と表裏一体）。

## 3. 期待値の現実（正直な前提）

- 日次・大型株のペア間 lead-lag は**あっても弱く不安定**（裁定で消えやすい）。
- 強く出やすいのは: (a) イントラデイ、(b) 大型→小型、(c) **サプライチェーン / 持株・同テーマでつながる銘柄**。
- 狙いに一番効くのは (c)。ここで **M3 の LLM が本領**を発揮する: ニュース・開示から「関係が深い銘柄群」の**関係グラフ**を作り、lead-lag 探索を意味のあるペアに絞る。これがこの企画の**本当の差別化点**になり得る。

## 4. 予測精度を上げる推奨（優先度: 大 → 小）

> 大前提: 日次・大型株の方向はほぼ効率的で、テクニカルの作り込みは収穫逓減。効くのは「測り方・当てる対象・情報源」を変えること。

### ① 確信度で“棄権”する（HOLD）＋確率較正 ★最優先・低コスト
- 全日当てにいかず**確信度上位だけ予測**（要件 §4 の HOLD）。予測した分の精度が大きく上がり実用的。
- lightgbm の出力確率はそのままだと当てにならないので**確率較正（Platt / isotonic）**とセット。
- 評価は「カバレッジ（予測割合）× その精度」のトレードオフ曲線で見る。

### ② 「絶対方向」→「相対（クロスセクション）」予測 ★効果大
- 市場全体の上下（日次ではほぼ予測不能）を外し、**同日内で“どれが相対的に強いか”**を当てる。
- 予測不能な共通要因（β）を除くので銘柄固有シグナルが学習しやすい。lead-lag の市場中立化（§2③）と同じ発想。

### ③ “新情報イベント”を主軸にする（M3 の本命）★差別化点
- 価格は織り込み済み。**決算・上方修正・開示（TDnet）**など新情報が入るイベント周辺で edge が出やすい。
- LLM で「関係グラフ＋インパクト」に変換し、イベント時にどの関連銘柄が翌日動くかを当てる。

### ④ 特徴量を厚くする（中）
- cross-stock ラグ（市場中立化残差）＝ lead-lag、セクター/ピア集約、市場レジーム（VIX 水準・トレンド）、決算近接、米国オーバーナイト（既存）。

### ⑤ データを増やす（中・地味に効く）
- 履歴 5 年＋、**銘柄数を増やす**（クロスセクション・lead-lag 探索ほど銘柄数が効く）。

### ⑥ モデル（小・投資対効果は低い）
- lightgbm 据え置き。`class_weight`、軽い early stopping、確率較正まで。ハイパラ探索に時間を溶かさない。

### 必須（“見かけの精度”に騙されない）
- 数字を上げる施策ではなく、**上がった数字が本物かを担保**する施策。欠くと過学習で精度を錯覚する。
- FLAT 閾値は**銘柄ボラ連動**、評価は **macro-F1**（accuracy 単独は FLAT 偏重で簡単に高精度に見える）。
- **日付境界 walk-forward / 3 ベースライン比較 / 有意性チェック**（STATUS に反映済み）。

---

## 5. もし 3 つだけ選ぶなら

1. **確信度で棄権（①）＋確率較正** — 実用精度を一番手早く上げる
2. **クロスセクション（相対）予測（②）** — 学習しやすい対象に変える
3. **イベント駆動の LLM 新情報（③）** — 価格だけでは出ない edge を入れる

> lead-lag（④）は「②③の枠組みの中の有力な特徴」と位置づけるのが、効果と現実性のバランスが良い。単独主役は日次・大型株では弱く不安定になりがち。
