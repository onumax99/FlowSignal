# FlowSignal 要件定義書

> 各種銘柄・市場の動きと時事ニュースから関連性を抽出し、翌営業日の株価方向を予測するアプリケーション。

- 作成日: 2026-06-15
- ステータス: PoC（プロトタイプ）フェーズ
- バージョン: 0.1（初版）

---

## 1. 概要

FlowSignal は、株価・マーケット指標といった**定量データ**と、時事ニュースから抽出した**関連性・感情**を組み合わせ（ハイブリッド）、対象銘柄の**翌営業日の値動きの方向**を確信度付きで予測する。LLM によるニュース分析を差別化点とし、予測の根拠を自然言語で提示する。

## 2. 目的・前提

| 項目 | 内容 |
|---|---|
| 対象市場 | 日本株（東証上場銘柄） |
| 主目的 | 個人投資の意思決定補助（自分用） |
| 予測手法 | ハイブリッド（時系列ML × LLMニュース分析） |
| 進め方 | PoC / プロトタイプから段階的に |
| 法的位置づけ | 個人利用を前提とし、投資助言業の登録対象外。他者提供時は金商法の検討が必須（→ 第11章） |

### 期待値の前提（重要）

翌日株価のピンポイント予測は本質的に困難（効率的市場仮説・高ノイズ）。PoC のゴールは「正確に当てる」ことではなく、**ベースライン（ランダム／常時上昇／前日同方向）を有意に上回る予測の傾きを抽出できるか**を検証することに置く。

## 3. スコープ

### PoC に含む
- 流動性の高い大型株 **10〜30銘柄**を対象
- 翌営業日の方向（3クラス）＋確信度の予測
- ニュースの銘柄関連付け・感情スコア化（LLM）
- バックテストによる評価とベースライン比較
- 予測根拠の言語化、簡易ダッシュボードでの提示

### PoC に含まない（将来検討）
- 当日値・イントラデイ（分足）予測、リアルタイム推論
- 自動売買・発注連携
- 多数ユーザー向け提供、課金、認証基盤
- 米国株・暗号資産・FX への拡張

## 4. 予測対象の定義

- **ターゲット**: 翌営業日終値の当日比方向を **3クラス分類**
  - `UP` … 翌日リターン > +閾値
  - `FLAT` … −閾値 〜 +閾値（横ばい）
  - `DOWN` … 翌日リターン < −閾値
  - 閾値は銘柄ごとのボラティリティに応じて設定（例: ±0.5%〜±1%。PoCで調整）
- **出力**: クラス＋**確信度スコア**（クラス確率）
- **棄権**: 確信度が一定未満の日は「予測しない（HOLD）」を許容し、実用度を高める

## 5. 機能要件

| ID | 機能 | 概要 |
|---|---|---|
| FR-1 | データ収集 | 株価OHLCV・マーケット指標・適時開示・ニュースを日次バッチで取得 |
| FR-2 | ニュース関連付け（LLM） | ニュース → 登場企業・テーマ抽出 → 対象銘柄へマッピング |
| FR-3 | ニューススコア化（LLM） | 感情（ポジ/ネガ）・インパクト・関連度を数値化し特徴量化 |
| FR-4 | 特徴量生成 | テクニカル＋マーケット＋LLM由来スコアを時点を揃えて結合 |
| FR-5 | 予測 | 翌営業日の方向（3クラス）＋確信度を出力 |
| FR-6 | 根拠提示（LLM） | どのニュース・指標が効いたかを自然言語で説明 |
| FR-7 | バックテスト/評価 | 過去データで的中率・収益性を検証、ベースラインと比較 |
| FR-8 | 提示 | 銘柄ごとの予測・確信度・根拠を一覧表示（Streamlit） |

## 6. データソース（無料中心）

| 種別 | ソース | 備考 |
|---|---|---|
| 株価 OHLCV | J-Quants API（無料枠）／補助 yfinance（`xxxx.T`） | 無料枠は当日値に遅延あり |
| マーケット指標 | 日経平均・TOPIX・USDJPY・米国指数(前日終値)・VIX・セクター指数 | |
| 適時開示 | TDnet（決算・業績修正・M&A 等） | 構造化されており有用 |
| ニュース | RSS（ロイター等）・Yahoo!ニュース 等 | 日経/Bloombergは有料。スクレイピングは各規約遵守 |

> **データリーク防止（必須）**: すべてのデータでタイムスタンプを厳守。ニュース公開時刻と取引時間の前後関係を守り、予測時点で入手不可能な未来情報を特徴量に混入させない。

## 7. システム構成（パイプライン）

```
[収集]   株価 / マーケット指標 / 適時開示 / ニュース（日次）
   ↓
[LLM分析] ニュース → 銘柄マッピング・感情・インパクト・関連度スコア
   ↓
[特徴量]  テクニカル + マーケット + LLMスコア を結合（時点を揃える）
   ↓
[予測]    LightGBM等で 翌日方向(3クラス) + 確信度
   ↓
[評価]    バックテスト（時系列split・リーク防止・ベースライン比較）
   ↓
[提示]    ダッシュボード（予測 + 確信度 + LLMによる根拠説明）
```

## 8. ハイブリッド予測の詳細

### 8.1 ML（定量）
- モデル: PoC は **LightGBM**（軽量・特徴量重要度が得られる）
- 特徴量カテゴリ:
  - テクニカル: リターン、移動平均、RSI、MACD、ボラティリティ、出来高変化
  - マーケット: 日経平均・TOPIX・USDJPY・前日米国指数・VIX・セクター指数
  - カレンダー: 曜日、月初/月末、決算期

### 8.2 LLM（関連性・感情）— 差別化点
- 利用モデル: Claude API（`claude-opus-4-8`／コスト重視は `claude-sonnet-4-6`）
- 役割:
  - ニュース本文から企業・テーマを抽出し対象銘柄へ関連付け
  - 感情（ポジ/ネガ）・インパクト・関連度を数値化 → ML特徴量へ
  - 予測結果の根拠を自然言語で説明
- スコアはタイムスタンプ付きで保存し、ML特徴量に時点整合で結合

## 9. 評価方法

- **分類精度**: 的中率、Precision/Recall、F1、AUC
- **金融的評価**: バックテストで 累積リターン・勝率・シャープレシオ・最大ドローダウン
- **ベースライン比較（必須）**: 「常に上昇」「ランダム」「前日同方向」に対して有意に上回るかを検証
- **検証手法**: 時系列を尊重した分割（walk-forward / time-series split）でリークを防止

## 10. 非機能要件

| 項目 | 方針（PoC） |
|---|---|
| 言語/基盤 | Python |
| 主要ライブラリ | pandas, LightGBM, pandas-ta, scikit-learn |
| データ取得 | J-Quants / yfinance / RSS |
| LLM | Claude API |
| データ保存 | parquet（特徴量）＋ SQLite（メタ・予測履歴） |

## 11. リスク・法務・免責

- **個人利用前提**のため投資助言業の登録は不要。将来、他者へ予測を提供する場合は**金融商品取引法（投資助言・代理業）の検討が必須**。
- アプリ内および出力に**免責文を明記**（投資は自己責任／予測の正確性・収益を保証しない）。
- 各データソースの**利用規約を遵守**（再配布・スクレイピングの可否）。
- 過学習・バックテストの楽観バイアスに注意（ベースライン比較を徹底）。

## 12. マイルストーン（PoC）

| ID | 内容 | 完了条件 |
|---|---|---|
| M1 | データ取得基盤 | 対象10〜30銘柄の株価＋ニュースを日次取得・保存できる |
| M2 | ベースラインML | テクニカルのみで方向予測し、ベースラインと比較評価できる |
| M3 | LLMニュース特徴量 | LLMスコアを追加し、M2比の改善幅を測定できる（PoCの肝） |
| M4 | バックテスト＋根拠説明 | 金融的評価指標を算出し、予測根拠を提示できる |
| M5 | ダッシュボード | Streamlitで銘柄別の予測・確信度・根拠を閲覧できる |
